# سلّمني — منصة سوق سوبر ماركت متعدّدة التجار (الخادم)

تنفيذ الـ Backend وفق الـ PRD v0.6. معمارية طبقية: **ASP.NET Core + EF Core + PostgreSQL**.

## البنية
```
src/
  Sallimni.Domain          الكيانات + الـ enums (لا اعتماديات)
  Sallimni.Application      المنطق النقي: محرّك التقسيم، الضريبة، ETA، الموجات
  Sallimni.Infrastructure   EF Core DbContext + الهجرات + خدمات التنسيق
  Sallimni.Api              المتحكّمات + DI + بيانات أولية + Swagger
```

## ما يعمل الآن (مُتحقَّق منه طرفياً)
- **محرّك تقسيم الطلب** (قلب النظام): إسناد كل صنف للتاجر الأرخص + **Fallback** عند النفاد + **لقطة سعر**.
- **الضريبة شاملة بالعرض**، مشتقّة في الفاتورة (شرائح ISTD: معفى/0/2/4/5/10/16).
- **العمولة** على الإجمالي شامل الضريبة (نسبة افتراضية + استثناء لكل تاجر).
- **ETA عند الطلب** (Wave-based) + الإلحاق التلقائي بأقرب موجة.
- **فحص السعر بالباركود** (Scan-to-Compare) — يُرجع أرخص سعر شامل الضريبة.

## التشغيل
متطلّبات: .NET SDK 9، PostgreSQL يعمل على `localhost:5432` (مستخدم `postgres`/`postgres`).

```powershell
cd "src\Sallimni.Api"
$env:ASPNETCORE_ENVIRONMENT="Development"
$env:ASPNETCORE_URLS="http://localhost:5080"
dotnet run --no-launch-profile
```
عند الإقلاع في وضع التطوير: تُطبَّق الهجرات تلقائياً وتُزرع بيانات تجريبية. Swagger على `http://localhost:5080/swagger`.

سلسلة الاتصال في `src/Sallimni.Api/appsettings.json` → `ConnectionStrings:Sallimni`.

## تطبيق الزبون (.NET MAUI)
`apps/Sallimni.CustomerApp` — منصّات الإصدار الأول: **Android + Windows** (iOS/Mac لاحقاً عند توفّر جهاز Mac).
معماريّة MVVM (CommunityToolkit.Mvvm)، هوية برتقاليّة، عربي/إنجليزي مع RTL حيّ.

**الشاشات (بأسلوب أسواق التوصيل):** شبكة الأصناف (فئات بأيقونات) ← منتجات الفئة (بطاقات بشارة **توفير** وسعر عادي مشطوب) ← صفحة تفاصيل المنتج (صورة كبيرة + وصف + إضافة للسلة) · فحص السعر بالباركود · السلّة + تأكيد الطلب · الطلب/التتبّع (ETA برتقالي) · الإعدادات.

> **التوفير:** يُحسب لكل منتج كنسبة بين **أرخص سعر** (سعرنا) و**أعلى سعر** متاح بين التجار — يُعرض كشارة "وفّر %" وسعر مشطوب، مجسّداً جوهر المنصّة.

**الرئيسية المصقولة:** ترويسة بالهوية + إجمالي السلّة + شارة **"توصيل مجاني"** (المنصّة تتحمّل كلفة التوصيل) · **شريط بحث** عن المنتجات · **شريط "أوفر العروض"** الأفقي (المنتجات الأعلى توفيراً) · شبكة الأصناف.
> نقاط نهاية الدعم: `GET /api/catalog/products?q=` (بحث) و`GET /api/catalog/offers` (أعلى توفير).

**مُتحقَّق منه:** يبني ويعمل على Windows، يحمّل الأصناف وأسعارها حيّاً من الخادم، RTL عربي يعمل.

تشغيل تطبيق الزبون (ويندوز، مع تشغيل الخادم):
```powershell
dotnet run --project "apps\Sallimni.CustomerApp" -f net10.0-windows10.0.19041.0 --no-launch-profile
```
> عنوان الخادم: ويندوز `http://localhost:5080`، ومحاكي Android `http://10.0.2.2:5080` (مضبوط في `MauiProgram.cs`).
> مسح الكاميرا الحيّ للباركود يُضاف على الأجهزة (مكتبة ZXing/ML Kit)؛ الشاشة حالياً تدعم الإدخال اليدوي + النتيجة.

## تطبيق التاجر (.NET MAUI)
`apps/Sallimni.MerchantApp` — Android + Windows، هوية **خضراء مزرقّة** (تمييز عن الزبون)، عربي/إنجليزي + RTL.

**الشاشات:** الطلبات الواردة (استقبال الطلبات الفرعية + بدء التحضير/جاهز) · المخزون والأسعار (ربط/تعديل سعر وكمية كل صنف + إتاحته) · إضافة صنف (طلب اعتماد للإدارة) · الإعدادات (اختيار المتجر + اللغة + حالة التسجيل الضريبي).

**نقاط نهاية الخادم:** `MerchantsController` — كتالوج التاجر، upsert السعر/المخزون، الطلبات الفرعية + تحديث الحالة، طلبات إضافة الأصناف.

**مُتحقَّق منه:** يبني ويعمل على Windows؛ شاشة الطلبات تعرض الطلبات الواردة حيّاً؛ مسارات الكتابة (تحديث الحالة، السعر/المخزون، طلب صنف) تعمل عبر الخادم.

تشغيل تطبيق التاجر (ويندوز، مع تشغيل الخادم):
```powershell
dotnet run --project "apps\Sallimni.MerchantApp" -f net10.0-windows10.0.19041.0 --no-launch-profile
```

## تطبيق الإدارة (.NET MAUI)
`apps/Sallimni.AdminApp` — Android + Windows، هوية **نيلية** (مالك المنصّة)، عربي/إنجليزي + RTL.

**الشاشات:** الاعتمادات (اعتماد/رفض طلبات أصناف التجار — الاعتماد يُنشئ بطاقة Product) · المهام (الموجات → إنشاء مهمة تجميع/توزيع + إسناد سائق + عرض المحطات) · التسويات (المستحق لكل تاجر بعد العمولة + وسم مُسوّى) · الإعدادات (نسبة العمولة + إعداد الموجات + اللغة).

**نقاط نهاية الخادم:** `AdminController` + `AdminService` — اعتماد الطلبات، إنشاء أصناف/تصنيفات، إنشاء مهام التوصيل من الموجات (محطة لكل متجر/زبون) + إسناد السائقين، ضبط العمولة والموجات، التسويات.

**مُتحقَّق منه:** يبني ويعمل على Windows؛ شاشة الاعتمادات تعرض الطلبات المعلّقة حيّاً مع اعتماد/رفض؛ كل مسارات الإدارة (اعتماد، إنشاء مهام تجميع+توزيع، إسناد سائق، تحديث العمولة، تسوية) مُتحقَّقة عبر الخادم.

تشغيل تطبيق الإدارة (ويندوز، مع تشغيل الخادم):
```powershell
dotnet run --project "apps\Sallimni.AdminApp" -f net10.0-windows10.0.19041.0 --no-launch-profile
```

## تطبيق السائق (.NET MAUI)
`apps/Sallimni.DriverApp` — Android + Windows، هوية **زرقاء**، عربي/إنجليزي + RTL.

**الشاشات:** مهامي (قائمة مهام التجميع/التوزيع المُسندة) ← تفاصيل المهمة (بدء، **مسح QR** لكل محطة: استلام من متجر / تسليم وتحصيل **COD** من زبون، إنهاء) · الإعدادات (اختيار السائق + اللغة).

**نقاط نهاية الخادم:** `DriverController` + `DriverService` — مهام السائق، بدء المهمة، مسح الاستلام (متجر→سائق) + التسليم للمستودع، التحميل + مسح التسليم (سائق→زبون) مع COD، إنهاء المهمة. كل مرحلة تُسجّل `ScanEvent` (سلسلة العهدة) وتنقل الحالات.

**مُتحقَّق منه:** يبني ويعمل على Windows؛ قائمة المهام تعرض المهام المُسندة حيّاً؛ تدفّق كامل (تجميع: بدء→استلام→تسليم مستودع؛ توزيع: تحميل→تسليم+COD→إنهاء) مُتحقَّق عبر الخادم مع **تسجيل مراحل QR الأربع** ووصول الطلب إلى "مُسلَّم ومحصَّل".

تشغيل تطبيق السائق (ويندوز، مع تشغيل الخادم):
```powershell
dotnet run --project "apps\Sallimni.DriverApp" -f net10.0-windows10.0.19041.0 --no-launch-profile
```

## تحسين المسار ووقت الوصول الدقيق (قسم 6.2) ✅
منطق نقي في `Sallimni.Application/Services`:
- [`GeoUtils`](src/Sallimni.Application/Services/GeoUtils.cs) — مسافة Haversine + زمن قيادة.
- [`RouteOptimizer`](src/Sallimni.Application/Services/RouteOptimizer.cs) — **الجار الأقرب (Nearest-Neighbor)** من المستودع، بلا أي API خارجي.

عند إنشاء مهمة التوزيع: تُرتَّب محطات الزبائن بالأقرب فالأقرب، ويُحسب **وقت وصول تقديري لكل محطة** (انطلاق الموجة + قيادة تراكمية + زمن تسليم)، ويُحدَّث **وقت تسليم كل طلب** من تقدير المتوسط لقيمة المسار الفعلية. يظهر الوقت في تطبيق السائق لكل محطة.

> مُتحقَّق: 3 طلبات لـ 3 عناوين → ترتيب محسّن بأوقات وصول متدرّجة (22:06 → 22:15 → 22:27).

## جاهزية النشر على المتاجر (قسم 15) ✅
- **حذف الحساب الفعلي:** `AccountController` (`DELETE /api/account/{customer|merchant|driver}/{id}`) يُجهّل البيانات الشخصية ويوقف الحساب (تطبيقاً لـ Apple 5.1.1 / Google). أزرار "حذف الحساب" في التطبيقات الثلاثة مربوطة فعلياً.
- **صفحات الويب العامّة:** `/privacy.html` · `/terms.html` · `/delete-account.html` (عربي/إنجليزي) تُخدَّم من `wwwroot`، وروابطها داخل إعدادات التطبيقات.

> مُتحقَّق: الصفحات تُخدَّم (200)، وحذف الزبون/التاجر/السائق يُجهّل الاسم ويزيل العناوين/يوقف الحساب.
